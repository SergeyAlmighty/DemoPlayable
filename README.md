# Playable Ads Test (Yonote)

## Выбранный движок и почему

Использован стек **Vite + TypeScript + vanilla JS (Canvas 2D)**.

- **Vite** выбран за быстрый dev-сервер, простую сборку и минимальный overhead для небольшого playable.
- **TypeScript** добавляет типобезопасность для игровой логики (юниты, состояние, input, рендер), что снижает риск регрессий.
- **Vanilla JS + Canvas** позволяет держать размер бандла низким, не тянуть UI-фреймворки и при этом получать стабильный FPS для простых 2D-механик.

## Как собрать и запустить локально

Требования:

- Node.js 20+
- npm

Команды:

```bash
npm install
npm run dev
```

После запуска откройте URL из терминала (обычно `http://localhost:5173`).

Production-сборка:

```bash
npm run build
npm run preview
```

## Какие оптимизации по размеру были применены

- Выбран **vanilla UI** без сторонних фреймворков и игровых движков, чтобы снизить итоговый размер JS.
- Визуалы сделаны процедурно через Canvas (градиенты/примитивы), без тяжёлых текстур и спрайтшитов.
- Логика собрана в один entrypoint без лишних зависимостей.
- Используется production-сборка Vite с минификацией и gzip-friendly output.

Текущий результат production-сборки:

- JS: ~7.6 KB (до gzip), ~3.3 KB (gzip)
- CSS: ~2.0 KB (до gzip), ~1.0 KB (gzip)
